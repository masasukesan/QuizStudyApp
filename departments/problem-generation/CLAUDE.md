# ■ 部署理念（最重要）

**「勉強が苦手な子でも、思考プロセスを理解しながら共通テストで高得点を取れるようにする。」**

そのために、  
**“答えそのものではなく、考え方・因果関係・途中式・根拠” を選ばせる問題** を最優先で作成する。

---

# ■ 最上位ルール：四択問題は正解が必ず1つ（絶対遵守）

> このルールは部署理念と同格の最上位仕様である。  
> いかなる問題においても、以下を違反してはならない。

**四択問題において、正解になりうる選択肢は必ず1つだけにする。**

### ● なぜ重要か

勉強が苦手な生徒が「自分の選んだ答えも正しいのでは？」と混乱すると、  
学習意欲が損なわれ、思考プロセスへの信頼が失われる。  
正解が1つに絞られていることは、学習アプリの最低条件である。

### ● 特に注意すべき場面（数学）

数学では「AとBをまとまりとして扱う方法」「残りを先に並べて隙間を数える方法」「全体から引いて求める方法」など、  
同じ答えに到達する複数の正しい解法が存在する。  
そのままでは複数の選択肢が正解になる。

### ● 対処法（どちらかを必ず行う）

① **解法を問題文に明示する**  
　→「AとBを1つのまとまりとして扱う方法で求める。正しい手順はどれか。」のように、  
　　どの解法を使うかを指定すれば、他の解法の選択肢は誤答になる。

② **選択肢に意図的な誤りを仕込む**  
　→ 正しい方向性だが「計算が間違い」「操作が足りない」「用語が違う」  
　　など、どこかに誤りを入れて1つに絞る。

### ● 禁止する問いかけの例

- 「最も効率的な方法はどれか」  
- 「最初に行う処理はどれか」  
- 「正しい考え方として適切なものを選べ」（解法複数成立の場合）

### ● 最終確認（問題生成後に必ず実施）

生成した4択を並べ、**自分が生徒なら「これも正しいのでは？」と思う選択肢がないか**を確認する。  
少しでも疑わしければ問題文か選択肢を修正する。

# ■ 方針（思考プロセス重視）

### ● 1. 答えそのものを問う問題は最小限にする

- 英単語・漢字など例外を除き、  
  **「どう考えるか」を問う問題を中心に作る。**

### ● 2. 選択肢は “思考の分岐” を表現する

- 正しい考え方
- よくある誤解
- 典型的なミス
- 条件の読み違い

これらを選択肢として提示する。

### ● 3. 中学生でも理解できる表現で作る

- 抽象的な表現は禁止
- 難しい記号・専門用語は避ける
- 数式はプレーンテキストで書く

### ● 4. 全教科に対応する

- 国語：根拠の位置・本文構造
- 数学：途中式・式の立て方
- 英語：根拠となる英文の要点
- 理科：実験の考察プロセス
- 社会：因果関係・理由付け

# ■ 数式表記ルール（数学科目のみ）

- 「^2」「^3」禁止 → **²、³** を使用
- 分数指数（有理数指数）の表記：`^(1/2)` `^(2/3)` などの形式は禁止 → **ᵐ/ⁿ** 形式を使用
  - 例：a^(1/2) → **a¹/²**、8^(2/3) → **8²/³**、x^(3/4) → **x³/⁴**
  - 使用する上付き数字：⁰¹²³⁴⁵⁶⁷⁸⁹⁻（Unicode 上付き文字）
- LaTeX 記法は禁止
- プレーンテキストで書く
- 中学生でも読める表記を優先
- 順列・組み合わせは **「nPr」「nCr」形式** を使用する（例：9P2、10C3）
- 「P(n, r)」「C(n, r)」形式は禁止（高校教科書・共通テストの表記に合わせる）
- 解法の俗称（例：かたまり法・隙間挿入法・補集合法）は使用禁止
  → 「AとBを1つのまとまりとして扱う」「残りを先に並べて隙間を数える」「全体から引いて求める」のように説明的な表現を使う
- **逆三角関数表記（arcsin・arccos・arctan）は絶対禁止**
  - 高校数学の学習指導要領の範囲外であり、共通テストにも出題されない
  - 代わりに以下の表現を使う：
    - `arcsin(x)` → `sinθ = x となる θ`、または `sin の値が x（鋭角・鈍角を確認）` 等
    - `arccos(x)` → `cosθ = x となる θ`、または `cos の値が x（鋭角・鈍角を確認）` 等
    - `arctan(x)` → `tanθ = x となる θ`、または `cosφ = a/R かつ sinφ = b/R を満たす φ` 等

# ■ 出力仕様（全教科共通）

問題は必ず以下の JSON 形式で出力する：

```json
{
  "id": "unique-id",
  "subject": "科目名",
  "unit": "単元名",
  "difficulty": "basic | standard | exam",
  "question": "問題文（思考プロセスを問う）",
  "choices": [
    { "label": "A", "text": "選択肢の文章" },
    { "label": "B", "text": "選択肢の文章" },
    { "label": "C", "text": "選択肢の文章" },
    { "label": "D", "text": "選択肢の文章" }
  ],
  "correct": "A",
  "reasoning_tags": ["考え方", "途中判断", "落とし穴"]  // ← 任意フィールド。省略可。
}
```

■ 選択肢の構造（重要）

- 選択肢 A/B/C/D は「位置」であり、意味を持たせない。

- 正答の文章は1つだけ作成する。
  （問題文の意図に沿った”正しい思考プロセス”を表現する）

- 誤答の文章は3つ作成する。
  （典型的な誤解・途中判断のズレ・落とし穴を表現する）

■ 選択肢のテキストに含めてはいけないもの（絶対禁止）

- **「（～な誤り）」「（～した誤り）」「（誤り）」「（間違い）」** などのメタ情報を
  選択肢の文末や括弧内に含めてはならない。
- 「（～の方法）」「（～のアプローチ）」のような **解法カテゴリを説明するラベル** も禁止。
- 選択肢は「生徒がどれを選ぶか」を問うものであり、
  選択肢自体が「これは誤りだ」と示唆する文言はヒントになり、学習効果を損なう。

【悪い例】
  “a₁=3, 公差=4 → aₙ = 3 + n×4 = 4n+3（n−1 を n にした誤り）”
  “S₂₀ = 20{2×2+(20−1)×3} = 1220（2で割るのを忘れた誤り）”

【良い例】
  “a₁=3, 公差=4 → aₙ = 3 + n×4 = 4n+3”
  “S₂₀ = 20{2×2+(20−1)×3} = 1220”

メタ情報は `explanations.json` の `common_mistakes` フィールドに記述すること。

- 4つの選択肢（正答1＋誤答3）をランダムに並び替える。

- 並び替え後の位置に応じて
  "correct": "A" | "B" | "C" | "D"
  を自動設定する。

- 正答・誤答の文章そのものはランダムにしてはならない。
  ランダム化するのは「選択肢の並び順」のみとする。

  ■ 問題ストック数（重要・絶対遵守）

- 問題生成部は、各単元につき **30問（basic:10問 ＋ standard:10問 ＋ exam:10問）** を作成する。
  - 難易度ごとにきっちり10問ずつ。過不足は一切認めない。
  - basic=9問・standard=11問など、合計30問でも配分が崩れている場合は作り直す。

- 30問の問題は、その単元から共通テストなどへ出題されている内容を網羅するものにする。

  ■ 既存問題との重複方針

- 共通テスト頻出パターンを全て網羅したうえで、**数値・状況・条件が異なれば多少の重複はOK**とする。
  - 例：「sin(α+β) を求める」という構造が同じでも、使う角度や与える条件が違えば別問題として扱う。
  - 逆に、問題文・数値・選択肢が実質同一の問題を複数混入させてはならない。
- 重複が懸念される場合は `reasoning_tags` を活用して出題の意図を差別化すること。

  ■ 正答・誤答の品質保証（必須）

- 正答の文章は、問題文の意図から逆算し、
  「最も正しい思考プロセス」を表現したものとする。

- 誤答の文章は、実際の学習者が陥りやすい
  「典型的な誤解」「途中判断のズレ」「落とし穴」
  のいずれかに必ず該当する内容とする。

- 正答・誤答の文章を生成した後、以下の整合性チェックを行う：
  1. 正答が問題文の意図と一致しているか
  2. 誤答が誤解・ズレ・落とし穴になっているか
  3. 誤答が正答と同じ意味になっていないか
  4. 正答が誤答のような曖昧表現になっていないか
  5. 4つの選択肢が同じ意味になっていないか
  6. **正解が1つだけか（→ 最上位ルール参照。疑わしければ問題文・選択肢を修正）**

- このチェックを通過した場合のみ、選択肢をシャッフルし、
  "correct" を設定する。

# ■ 教科別：思考プロセスの問わせ方

### ■ 国語

- 「本文のどこが根拠か？」
- 「筆者の主張はどれか？」
- 「なぜその心情になるのか？」

### ■ 数学

- 「どの式を立てるべきか？」
- 「途中式として正しいのは？」
- 「どの条件を使うべきか？」

### ■ 英語

- 「どの英文が根拠か？」
- 「話者の意図は？」
- 「この単語の意味を推測する理由は？」

### ■ 理科

- 「実験結果から何が言える？」
- 「どの法則を使うべき？」
- 「なぜその変化が起きる？」

### ■ 社会

- 「なぜその出来事が起きた？」
- 「どの要因が影響した？」
- 「因果関係として正しいのは？」

---

# ■ 禁止事項

- 答えだけを選ばせる単純問題を量産しない
- 過去問の文章・図表を模倣しない
- 他の資格試験の構造を混入させない
- 難しすぎる抽象表現を使わない

---

**「問題を解くたびに “考え方” が身につき、  
勉強が苦手な子でも共通テストで高得点を取れるようにする。」**

これを最優先に、すべての問題を作成する。
