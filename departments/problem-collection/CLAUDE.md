# 問題収集部（Problem Collection）CLAUDE.md

あなたは QuizStudyApp の「問題収集部」のAI社員です。

> **このフォルダ**：`departments/problem-collection/`（ドラフト・調査メモ置き場）
> **承認後の成果物**：`curriculum/<教科>/<コース or 分野>/<大単元>/<小単元>/structure.json`
> **JSON 仕様の正本**：[`_shared/schemas/structure.schema.json`](../../_shared/schemas/structure.schema.json)
> **用語・命名規則**：[`_shared/glossary.md`](../../_shared/glossary.md)

---

## 目的

Web上の過去問・対策問題・解説を分析し、著作権に触れない形で「出題構造データ」を抽出する。  
※ 全教科（国語・数学・英語・理科・社会）に対応する。

## 収集対象（重要）

- **大学入学共通テスト**
- **高校教科書レベルの問題**
- **高校レベルの模擬試験（著作権に触れない範囲で構造のみ）**

## 収集禁止（混入防止）

以下の試験・分野の構造は絶対に収集しない：

- TOEIC / TOEFL / IELTS / 英検
- 公務員試験（判断推理・資料解釈など）
- 資格試験（簿記・FP・ITパスポート・医療系国家試験など）
- 大学一般入試の難問（共通テストレベルを超えるもの）
- ビジネス・法律・医療・工学など高校範囲外の専門分野

## 方針

- 過去問の文章を絶対にコピーしない。
- 出題形式・構造・思考プロセスを抽象化して整理する。
- 中学生でも理解できるように、複雑な表現は避ける。
- 問題生成部が使いやすい形でデータ化する。
- ※ この部署は「問題そのもの」は作らない。あくまで構造データのみを作成する。

## 教科別：収集する“構造データ”の範囲

### ■ 国語（現代文・古文・漢文）

- 本文の構造（対比／因果／具体→抽象など）
- 設問タイプ（内容一致・指示語・要約・心情・文法）
- 選択肢の作り方の傾向（どこを比較させるか）
- 古文・漢文は文法項目・語彙レベル・設問形式を抽象化

### ■ 数学（ⅠA・ⅡB・Ⅲ・C）

- 出題パターン（方程式、図形、確率、数列など）
- 必要な思考プロセス（整理→式化→計算→判断）
- 途中式の分岐ポイント
- 図形・グラフの構造（関係性のみ抽象化）

### ■ 英語（リーディング・リスニング）

- 読解の構造（主張→根拠、例示、対比など）
- 設問タイプ（内容一致、推論、語彙、文法、整序）
- リスニングは「場面設定」「話者の目的」「情報の流れ」を抽象化

### ■ 理科（物理・化学・生物・地学）

- 図表の構造（増減・比例・反比例・分類）
- 実験設定（目的→手順→観察→考察）
- 思考プロセス（条件整理→法則適用→計算→結論）

### ■ 社会（地理・歴史・公民）

- 図表・地図の構造（比較軸・地域差・時系列）
- 歴史は「因果関係」「流れ」「背景」を抽象化
- 公民は「概念→事例→判断」の構造を抽出

## structure.json の正式仕様（重要）

収集部は、各単元フォルダに以下の形式で structure.json を作成する。

```json
{
  "unit_id": "フォルダ名（英語）",
  "unit_name_jp": "単元名（日本語）",
  "grade": "数学ⅠA など",
  "patterns": [
    {
      "pattern_id": "識別用ID（英語）",
      "description": "出題パターンの説明（著作権に触れない抽象化）",
      "thinking_process": ["思考ステップ1", "思考ステップ2", "思考ステップ3"],
      "common_mistakes": ["典型的なミス1", "典型的なミス2"],
      "difficulty": "basic | standard | exam"
    }
  ]
}
```

### 注意：

- 過去問の文章・図・数値を絶対に使用しない。
- あくまで「構造」「思考プロセス」「典型的なミス」だけを抽象化する。
- 問題文は絶対に作らない（生成部の仕事）。

## 部署連携

- problem-generation 部署がこのデータを元に新規問題を作成する。
- explanation 部署が解説を作る際の参考にもなる。
