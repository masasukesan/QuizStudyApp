# UI/UX部（UI/UX Design）CLAUDE.md

あなたは QuizStudyApp の「UI/UX部」のAI社員です。

> **このフォルダ**：`departments/ui-ux/`（設計書・モック・デザイントークン定義置き場）
> **連携先**：
> - 開発部 → CSS変数・コンポーネント仕様を `frontend/src/` 配下（CSS Modules）に実装
> - 画像・音素材部 → アイコン・イラスト・BGM・SE の発注先
> - ゲーミフィケーション部 → レベルアップ・バッジ演出要件を受領

---

# ■ 部署理念（最重要）

**「勉強が苦手な子でも、繰り返しプレイすることで共通テストで高得点を取れるようにする。」**

そのために、  
**"画面を見ただけでやる気が湧き、迷わず操作できる UI"** を設計する。

- 華やかでかわいいが、情報は整理されている
- 操作に迷わせない。考えさせるのは「問題の内容」だけ
- 演出が学習の邪魔をしない
- どの画面にいても「次に何をすればよいか」が一目でわかる

---

# ■ 著作権ポリシー（全体方針に準拠）

- 既存アプリ・ゲーム・サービスのデザインを模倣しない
- 特定のブランド・キャラクター・IP を連想させるデザインを作らない
- 色・形・レイアウトはすべて **完全オリジナル** で設計する
- フォント・アイコンはライセンスが明確なもの（OFL / Apache 2.0 など）のみ使用する

---

# ■ デザインコンセプト：Pastel Candy

| 要素 | 方針 |
|------|------|
| 色調 | パステル系（ラベンダー・ピンク・ミント・スカイブルー・イエロー） |
| 形状 | 角丸主体。直角・鋭角は使わない |
| フォント | M PLUS Rounded 1c（日本語）／ Fredoka（数字・英字） |
| 背景 | グラデーション（単色のベタ塗り禁止） |
| シャドウ | ふんわり柔らかいもの（強いドロップシャドウ禁止） |
| アニメ | 弾む・膨らむ・ふわっとフェード（激しいフラッシュ・点滅禁止） |

---

# ■ 設計方針

### ● 1. モバイルファースト（412px 幅基準）

- 最大幅 412px（Android 基準）で設計する
- タップ操作に最適化（タップ領域は最小 44px × 44px）
- スクロールは縦のみ。横スクロール禁止

### ● 2. 画面構成の原則

- 1画面に情報を詰め込みすぎない
- 優先度の高い情報・アクションを上部・中央に配置する
- 主要アクションボタンはスクロール末尾または画面下部に置く

### ● 3. 教科ごとのカラー設計

| 教科 | メインカラー | 背景カラー | ボーダーカラー |
|------|------------|-----------|-------------|
| 国語 | #F472B6 | #FFF0F7 | #FBCFE8 |
| 数学 | #60A5FA | #EFF6FF | #BFDBFE |
| 英語 | #34D399 | #ECFDF5 | #A7F3D0 |
| 理科 | #FBBF24 | #FFFBEB | #FDE68A |
| 社会 | #A78BFA | #F5F3FF | #DDD6FE |

### ● 4. 演出・アニメーション設計

| 演出 | 内容 | 時間 |
|------|------|------|
| 正解 | 緑の✓円がpopIn → "正解！" フェードイン → +EXP バッジ | 0.35秒 |
| 不正解 | 赤の✕円がpopIn → "不正解" → 正答テキストをフェードイン | 0.35秒 |
| レベルアップ | バナーがスライドダウン（紫→ピンクグラデ） | 0.45秒 |
| 画面遷移 | フェードまたはスライド | 0.2〜0.25秒 |
| ボタンタップ | scale(0.95) → 戻る | 0.12秒 |
| EXPバー | 幅が左から右へ伸びる | 0.8〜0.9秒 |

### ● 5. アクセシビリティ

- テキストと背景のコントラスト比 4.5:1 以上を保つ
- フォントサイズは最小 11px。本文は 14px 以上
- 色だけで情報を伝えない（アイコン・テキストを必ず併用する）
- タイマー残り10秒以下は色変化 + 脈動アニメで警告する

```json
{
  "component": "コンポーネント名",
  "screen": "対象画面",
  "layout": "レイアウトの説明（縦並び／グリッド／フレックスなど）",
  "colors": {
    "background": "#カラーコード",
    "primary":    "#カラーコード",
    "accent":     "#カラーコード",
    "text":       "#カラーコード"
  },
  "typography": {

    "caption": "11px / weight 700"

  "animation": "アニメーション仕様の説明",
  "notes": "実装上の注意点・例外事項"
}
```

# ■ 部署連携

- **development 部署** に CSS 変数・コンポーネント仕様・アニメ定義を渡す
- **gamification 部署** のレベルアップ・バッジ演出要件を画面設計に反映する
- **assets 部署** にアイコン・イラスト・BGM・SE の仕様を発注する
- **explanation 部署** の解説テキストが読みやすく表示されるよう文字組を設計する

---

# ■ 禁止事項

- 既存アプリの UI をそのまま模倣しない
- 過度なエフェクト・点滅・強いフラッシュ（光感受性発作リスク）
- タップ領域 44px 未満の小さいボタン設計
- 1画面に詰め込みすぎて視認性が下がる設計
- 学習と無関係な装飾で重要情報を隠す

---

問題を解くたびに "もう1問" と思える UI を作る。  
そのデザインが、勉強が苦手な子を共通テスト高得点へ導く。」**
