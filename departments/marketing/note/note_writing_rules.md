# note記事 執筆ルール

最終更新：2026-06-29

---

## ■ バズり記事の構成を借りるときのルール

noteで継続的に読まれる記事の構成パターンを「借りる」ことで効率よく書く。
ただし以下を厳守する：

- **構成（骨格）だけ借りる** — タイトル・テーマ・内容・具体例はすべてオリジナル
- **参考にした記事は記録しない** — 借用元を明記すると印象が悪くなる
- **パターンが露骨にならないよう変形する** — 構成が同じでも口調・順序・見出しで差別化

---

## ■ 効果的な記事構成（バズ分析から抽出）

### フック（冒頭）
- 「読者が心の中で持っている問い」を1行で提示する
- 即座に結論を出す（まさくんスタイル）→「実は〇〇だった」という反転が続く
- 「これだけで〇割は終わり。でも読んで」という引き留め文を添えると離脱を防ぐ

### 本文の流れ
1. **なぜこの話をするか** — 塾・生徒・現場の体験から入る。数字（10年以上、何百枚）を入れると信頼感が増す
2. **読者が誤解していること** — 「〇〇と思っているかもしれないが、実は〜」の構造
3. **本当の原因・構造** — 数学的・教育的な構造説明。ここがまさくんの強み
4. **体験談ブロック** — 具体的な生徒の話（点数・期間・何が変わったか）。特定できない範囲で
5. **「量だけではダメ」ブロック** — よくあるアドバイスへの反論。共感を生む
6. **具体例** — 単元を絞った短い例示
7. **今日からできること3つ** — 保存欲を生む。動詞始まり、1〜2文で完結

### まとめとCTA
- まとめは2〜3行で完結。記事タイトルの問いに答える形
- CTAは「MATHACA（マサカ）では〜」で自然につなぐ。効果保証はしない

---

## ■ 文体・口調ルール

- **結論を先に出す** — 「〇〇だ。」で始めて後から展開
- **短文を多用** — 1文は20〜40字以内が理想。長い文は分割する
- **問いかけを使う** — 「おかしくない？」「なぜか。」で読者を巻き込む
- **「〜だ。」「〜なんだ。」** — 断定口調で信頼感を出す
- **「ぼく」** — 一人称はひらがな「ぼく」
- **改行は多め** — スマホ読みを想定。3〜4行で1段落が上限

---

## ■ テーマ選定ルール

参照：`movies/shorts/viral_concepts.md`

- viral_conceptsの★★★から選ぶ（共感系・挑戦系・意外性系）
- 既存のscript_00X.mdと単元が被らないか確認する
- 既存のnote記事と重複しないか確認する
- 「数学の構造的理解」が語れるテーマを優先（単なる問題解説ではなく）

優先度の高いテーマ候補（未使用）：
- V03：99%の高校生が最初に間違える計算（符号ミス・展開）
- V09：なぜ負×負=正なのか
- V11：二次方程式を暗算で解く方法
- V13：場合の数を余事象で激速にする
- V16：sinとtanを混同する勘違い

## ■ ファクトチェックチェックリスト（毎回必須）

記事を書いたら以下を確認してから保存する：

### 数学・教育的な事実
- [ ] 試験時間は正確か（共通テスト数学ⅠA = 70分、数学ⅡBC = 70分）
- [ ] 公式・定義の記述に誤りはないか
- [ ] 「型は〇〇種類」など数値の根拠はあるか（「くらい」「おおよそ」で濁すのはOK）
- [ ] 類比・例え（将棋など）が誤解を生まないか

### 表現・法的リスク
- [ ] 効果の断定がないか（「必ず〇〇になる」「絶対に上がる」はNG）
- [ ] 生徒のエピソードが特定の個人を特定できない範囲か（名前・学校名NG）
- [ ] 体験談を「全員に起きること」のように見せていないか
- [ ] 他社・他サービスとの比較がないか
- [ ] 架空の話を実話として書いていないか（エピソードは実体験の範囲内で）

### note規約・著作権
- [ ] 他の記事・書籍の文章を引用していないか
- [ ] 画像を使う場合は権利確認済みか

